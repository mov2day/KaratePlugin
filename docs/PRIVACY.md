# Privacy and Data Sharing - Karate DSL Generator

## Overview

This document explains what data the Karate DSL Generator extension shares with external services, specifically GitHub Copilot.

## GitHub Copilot Integration

### When is Data Sent?

Data is sent to GitHub Copilot **ONLY** when:
1. The setting `karateDsl.useCopilot` is set to `true` (default: `false`)
2. You run a test generation command
3. GitHub Copilot is available and authenticated

### What Data is Sent to Copilot?

#### ✅ Data That IS Sent

1. **Generated Karate Test Content**
   - The `.feature` file content that was generated by the extension
   - This is the test code you would see in the output file
   - Example: Scenario definitions, Given/When/Then steps, test data

2. **Full Source Context** (for better AI enhancement)
   - **Full OpenAPI Specification**: Complete JSON/YAML content of the API spec
   - **Full Confluence Page Content**: Complete HTML/text content of the page
   - **Extracted Requirements**: Parsed requirements and test cases from Confluence

3. **Context Metadata**
   - Brief descriptive strings such as:
     - `"OpenAPI spec: petstore, 5 endpoints"`
     - `"Confluence page: User Stories, 3 scenarios"`

**Purpose**: Sharing full context allows Copilot to generate more accurate and relevant test enhancements based on the actual API specification and business requirements.

#### ❌ Data That is NOT Sent

- ❌ **API credentials** (Confluence API tokens, OAuth tokens)
- ❌ **Authentication headers or secrets**
- ❌ **Actual API responses** from production systems
- ❌ **File system paths** or workspace information
- ❌ **User personal information**
- ❌ **Production data or real user data**

### Code Example

Here's exactly what gets sent from the code:

```typescript
// From: src/commands/generateFromOpenAPI.ts (line 68)
const context = `OpenAPI spec: ${specFileName}, ${endpoints.length} endpoints`;

// From: src/services/copilotService.ts (lines 42-48)
const messages = [
    vscode.LanguageModelChatMessage.User(
        `You are an expert in Karate DSL test framework...
        
        Context: ${context}  // <- Only this metadata
        
        Current Feature File:
        \`\`\`gherkin
        ${featureContent}  // <- The generated test (not the OpenAPI spec)
        \`\`\`
        
        Please enhance this Karate test by...`
    )
];
```

## Confluence Integration

### What Data is Sent?

When fetching from Confluence:
- **Sent to Confluence API**: Page ID or URL (to fetch the page)
- **Sent to Copilot** (if enabled): Generated test content + metadata like `"Confluence page: [title], [scenario count] scenarios"`
- **NOT sent to Copilot**: Raw Confluence page HTML, attachments, or full content

### Authentication

- Confluence API token is stored securely in VS Code's SecretStorage
- Token is never sent to Copilot
- Token is only used for Confluence REST API calls

## Third-Party Services

### GitHub Copilot
- **Purpose**: AI-powered test enhancement
- **Data Sent**: Generated test content + minimal metadata
- **Privacy Policy**: https://docs.github.com/en/site-policy/privacy-policies/github-copilot-privacy-statement
- **Opt-in**: Disabled by default

### Confluence (Atlassian)
- **Purpose**: Fetch test requirements from pages
- **Data Sent**: API authentication, page ID
- **Privacy Policy**: https://www.atlassian.com/legal/privacy-policy
- **Opt-in**: Only when using Confluence features

## Recommendations

### For Sensitive/Proprietary Projects

1. **Disable Copilot Enhancement**
   ```json
   {
     "karateDsl.useCopilot": false
   }
   ```

2. **Use Workspace Settings**
   - Configure per-project in `.vscode/settings.json`
   - Keep Copilot disabled for sensitive workspaces

3. **Review Before Enhancement**
   - Generated tests are created locally first
   - You can review them before enabling Copilot

4. **Sanitize Test Data**
   - Extension generates example data (not real data)
   - Review generated tests for any sensitive information

### For Public/Open Source Projects

- Copilot enhancement is safe to use
- Generated tests contain only example data
- No production credentials or real data included

## Extension Telemetry

This extension does **NOT**:
- ❌ Collect usage analytics
- ❌ Send telemetry data
- ❌ Track user behavior
- ❌ Store data on external servers
- ❌ Share data with third parties (except Copilot/Confluence when explicitly enabled)
