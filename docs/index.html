Automated Scenario Outlines & Test Data

Testing an API is more than checking a 200 OK. Our smart engine analyzes OpenAPI property fields to inject realistic synthetic data based on regex constraints, patterns, and property names (e.g. producing valid UUIDs, timestamps, and emails).

It automatically unrolls endpoints into Data-Driven Scenario Outlines covering positive execution paths, negative error states (400, 401, 404), and exact boundary specifications.

// Example Outline Generated by Engine

Scenario Outline: User Login Verification
    Given path 'users/login'
    And request { "email": "<email>", "password": "<password>" }
    When method POST
    Then status <status>

    Examples:
    | email             | password | status |
    | admin@company.com | valid123 | 200    |
    | badformat         | valid123 | 400    |
    | user@company.com  | wrong    | 401    |

Automatic API Spec Synchronization

When engineering updates the Swagger definition, your tests shouldn't fall behind. The plugin listens for modifications to your workspace OpenAPI definition files.

Upon saving, it automatically triggers an impact analysis to determine which feature files need modification to match new schemas. It uses AI to append new endpoints or rewrite modified structures while explicitly preserving manual assertions you have written previously.

  • 1. Detects change in `users-v2.yml`
  • 2. Analyzes impact on `users.feature`
  • 3. Intelligently updates request schema using Copilot
  • 4. Retains custom logic such as specific `.env` variables
